Instructions

You will build the five-stage pipelined MIPSL processor in verilog, which is a 16-bit version of the pipelined MIPS described in the lectures and textbook (Chapter 4).  It has the same instruction set as the single cycle MIPS-L of Homework 9.  The components such as the ALU is the same as in Homework 9.  Note that all arithmetic and logic operations are done in the ALU with the exception of sign extension, shifting, and address addition (for branch address calculation).  Your computer will eventually implement all the instruction of the MIPS-L of Homework 9 except the j, jal and jr instructions.

The are two parts to this project that leads to 5 subprojects.  (Note that the subprojects are a continuum of tasks for each Part.  Having subprojects makes it clearer to assign grade points.)

Information:

  • Description of the PMIPS-L0 and PMIPS-L processors. 

Part 1:  Implement PMIPS-L0.  This is a simplified version of the pipelined MIPS-L.  It has five stages of pipelined datapath but does not allow instructions to be pipelined.  In other words, only one instruction is processed through the computer at a time.  This means that the processor will have bubbles in it as well as stalls for instruction fetching.

Subproject 0:  For this subproject, you can download verilog implementations of components of the computer:  data memory and IO, register file, multiplexers (2:1 and 4:1), and ALU.  Here is the download in a zipped folder.  The folder includes the verilog modules in the MIPS-Parts.V file.  There are also three testbenches to test the modules, and an output trace file (word doc).  There is also a instruction memory file IM1.V.  This will be used in Subproject 1.  Notice it is a simple program that multiplies two numbers.  For this subproject, look over the modules and test them using the testbenches.  Also read the description of PMIPS-L0 and PMIPS-L (under "Information" above).

Subproject 1:  Implement the PMIPS-L0 in verilog to execute the following instructions:  addi, R-type, and beq.  Demonstrate your verilog code using veriwell.  It will run a simple program which multiplies two numbers.  Downloads will be ready November 4.

  • Download for control modules (zipped file):  This has modules for a controller and ALUcontrol.  The controller is incomplete and the ALUcontrol hasn't been tested and may be buggy.  There is a README file which you should read first.
  • Download for PMIPSL0 (zipped file):  This has an incomplete version of PMIPSL0 and a testbench.
  • Download for Subproject1 (zipped file).  There is a simple program in file IM1.V of multiplying two numbers.  It also has a testbench and a pdf file which discusses implementation.  Demonstrate to the TA for credit.

Subproject 2:  Implement the PMIPS-L0 in verilog to execute the instructions in Subproject 1 as well as lw, and sw.  Demonstrate your verilog code using veriwell.

  • Download for Subproject2 (zipped file).  Demonstrate to the TA for credit.

Subproject 3:  Implement the PMIPS-L0 described in Subproject 2 on the FPGA of the Digilent Basys  Board. 

  • Download for Subproject3 (zipped file).  Demonstrate to the TA for credit.

Part 2:  Implement the pipelined MIPS-L, which we will refer to as PMIPS-L.  It has five stage of pipelined datapath.  This processor will have simple branch-untaken branch prediction, so it will have the ability to flush the pipeline.  It will also check consecutive instructions to determine when it can insert bubbles.

Subproject 4:  Implement the PMIPS-L in verilog.  Demonstrate your verilog code using veriwell by having your computer run program IM1.V from Subproject 1.

Subproject 5:  Implement the PMIPS-L in verilog and on the FPGA of the Digilent Basys Board. Have your computer run program IM3.V from Subproject 3.
